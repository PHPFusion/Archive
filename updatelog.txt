PHP-Fusion 1.31 Update Log

30.07.2003
 Complete rewrite of the News system. In previous versions, news was stored in
text files, whilst the news info such as Subject, Date and Author was stored
in the database. Now the entire news item is stored in the database, this has
been done to allow anew feature. Your News article can be shown on one page,
with an extended version on another page. This feature does not have to be
used, but it provides a way of keeping your main news page uncluttered.
 Database Articles changes: added reads (smallint(6)) column to log how many
people have read it.
 Database News changes: added news (text), extendednews
(text) and reads (smallint(6)) columns.

03.08.2003
 Changed the theme template to a more minimalistic style for faster page loading.
 Added Total Members and Newest Member info to Users Online box.
 Added new layer of code to fix annoying multiple users online bug.

05.08.2003
 Changed site layout to fixed-width optimised for resolutions of 1024x768 and above.
 Updated the Upgrade script to update myCMS 1.3 to PHP-Fusion 1.31.

06.08.2003
 Reverted back to non fixed width site layout.
 Added Theme option to Admin Settings page to enable easy theme switching.
 Database Settings changes: added theme (varchar(32)) column.

07.08.2003
 Altered subheaders to support new switchable themes.
 Optimised Style Sheet.
 Added Hottest Threads and Latest articles to sidebar.

08.08.2003
 Added two new settings; Allow Visitor Shoutbox Post (Yes or No) and Allow
Visitor Comments (Yes or No). The latter of these two new settings affect
both News & Articles.
 Database settings changes: added visitorshoutbox (varchar(3)) and
visitorcomments (varchar(3))
 Added 2 new sidebar elements: Hottest Threads shows the 5 most popular
Forum Threads with post counts shown in brackets. Latest Articles shows
the 5 most recent Articles, these include new and updated Articles.
 Removed Google Websearch from sidebar.
 Default theme Aquarium completed.
